# 02. 核心概念

## 1. Bundle 是什么

Bundle 是 Jito 学习里最重要的概念之一。

官方文档给出的关键信息可以概括为：

- bundle 是一组交易
- 最多可包含 5 笔交易
- 按顺序执行
- 原子执行
- 全成全败

这意味着如果你的策略需要下面这种流程：

1. 先买入
2. 再调用某个程序
3. 最后卖出或付 tip

那么就可以把这些交易打成 bundle，避免只执行了一半。

## 2. Bundle 为什么重要

普通单笔交易适合简单动作。
但一旦策略依赖多步组合，单笔交易往往不够。

例如：

- 原子套利
- 清算 + 对冲
- 多步 DeFi 路由
- 保护某个关键交易优先落地

这时 bundle 的原子性非常关键。

## 3. Block Engine 在做什么

Block Engine 可以理解为一个高性能的交易与 bundle 入口层。

它主要做几件事：

- 接收 searchers、bots、dApps 提交的交易和 bundles
- 对 bundles 做模拟
- 对候选 bundles 做排序和筛选
- 把更有价值的结果发给 validator

从学习角度，你可以先把它理解成：

“一个帮助高竞争交易流更快、更有序进入验证者的执行层”

## 4. Auction 是什么

Jito 文档里提到 bundles 会进入拍卖流程。
原因很简单：区块空间有限，机会有限，冲突账户也有限。

官方文档提到的几个重点：

- 拍卖是为了最大化区块中的收益组合
- 会根据账户锁冲突关系组织局部拍卖
- 并行拍卖会以很短的 tick 周期运行
- 更高价值的组合更容易被选中

所以从工程角度看，bundle 不是“发出去就一定上链”，它是在参与竞争。

## 5. Tip 是什么

Tip 可以理解为开发者或搜索者为了提高 bundle / transaction 竞争力而支付给 validator 的额外激励。

学习时要区分两个概念：

- Priority Fee：Solana 原生的优先费
- Jito Tip：Jito 路径里的额外激励

在拥堵或竞争激烈的时候，只设置其中一个往往不够。

## 6. Jito 发交易和普通 RPC 发交易的区别

普通 RPC 路径更通用，适合常规 dApp。

Jito 路径更适合：

- 低延迟交易
- 需要 bundle 的场景
- 更强调落地速度和竞争力的机器人
- 希望减少部分 MEV 风险的发送方式

但代价是：

- 你需要理解更多系统行为
- 你需要管理 tip / fee / bundle 策略
- 你要面对拍卖与竞争失败的问题

## 7. ShredStream 为什么重要

很多 Solana 策略的竞争优势来源不是“更聪明”，而是“更快知道链上发生了什么”。

ShredStream 的意义就在这里：

- 更快收到区块更新相关数据
- 降低远程服务器接收数据的延迟
- 帮助交易系统更早进入下一步决策

对于高频系统，这往往是基础设施优势的一部分。

## 8. Jito-Solana 与 validator

Jito 文档把 Jito-Solana 描述为修改版 validator client。
它不是一个普通前端 SDK，而是验证者侧的重要基础设施。

开发者不用一开始就研究 validator 源码，但至少要理解这件事：

Jito 不是单纯的 API 服务，它背后连接的是 validator 执行与收益分发体系。

## 9. JitoSOL 是什么

如果从产品视角看 Jito，JitoSOL 是 Jito 非常重要的另一条线。

它是 Jito 的 liquid staking token。
用户质押 SOL 后可以拿到 JitoSOL，在保留流动性的同时获取：

- 常规质押收益
- 与 Jito 相关的 MEV 收益增量

所以你会看到 Jito 既是交易基础设施提供者，也和质押收益产品有关。
