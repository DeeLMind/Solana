# 扩展功能

来源：

- https://solana.com/zh/docs/tokens/extensions

## 这部分是什么

扩展功能主要对应 Token-2022 的能力增强。  
可以理解为：在基础 SPL Token 之上，为 mint 或账户增加额外规则。

官方侧边栏列出的重点扩展包括：

- 元数据与元数据指针扩展
- 不可转让代币
- 转账费用
- Memo 转账
- 不可变所有者
- 计息代币
- 默认状态
- 永久委托
- 关闭 Mint
- Token 组和成员
- CPI Guard
- 可暂停铸造

## 为什么需要扩展

传统 Token Program 功能比较纯粹：

- 铸造
- 转账
- 销毁
- 授权

但很多业务需要更细规则，比如：

- 每次转账自动扣手续费
- 某类票券不可转让
- 某些账户默认冻结
- 代币需要带内嵌元数据

## 例子 1：不可转让代币

适合场景：

- 学历证书
- 会员身份
- KYC 凭证

设计目标是：

- 用户能持有
- 但不能像普通代币那样自由转给别人

## 例子 2：转账费用

适合场景：

- 平台代币税费
- 国库抽成
- 自动手续费模型

每次转账时，程序会根据规则抽取一定费用。

## 例子 3：Memo 转账

适合场景：

- 充值备注
- 订单号绑定
- 风控审计

你可以把它理解为：

- 转账时强制带上一段附加说明

## 例子 4：CPI Guard

这是偏安全方向的扩展。  
当你的代币账户不希望被任意 CPI 间接操作时，这类防护就很有价值。

## 选型建议

如果你的代币只是：

- 普通积分
- 普通治理票
- 普通转账资产

先从基础 Token Program 开始就够了。

如果你需要：

- 强业务约束
- 安全隔离
- 链上规则更丰富

再考虑 Token-2022 扩展。

## 一个决策思路

```text
只是发币 + 转账 -> 基础 Token
需要手续费 / 不可转让 / 状态控制 -> 看 Token-2022 扩展
需要名称图片等展示信息 -> 看元数据方案
```
