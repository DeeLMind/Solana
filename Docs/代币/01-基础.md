# 代币基础

来源：

- https://solana.com/zh/docs/tokens
- https://solana.com/zh/docs/tokens/basics

## Mint Account

Mint Account 是某个代币的全局定义。

你可以把它理解为“币种说明书 + 全局计数器”。

它通常关心这些信息：

- 总发行量
- 小数位数
- mint authority
- freeze authority

## Token Account

Token Account 用来记录某个 owner 对某个 mint 的持仓数量。

核心字段可以简单理解为：

- 这是谁的账户
- 持有哪种代币
- 持有多少

## ATA（关联代币账户）

ATA 是某个：

- owner
- mint

对应的默认 token account 地址。

它的好处是：

- 地址可预测
- 钱包和应用更容易定位用户的默认持仓账户

## 例子：给用户发一个新代币

场景：

你要给用户发 `POINT` 积分币。

步骤：

1. 创建 mint
2. 为用户创建 ATA
3. 铸造 token 到用户 ATA

逻辑上就是：

```text
POINT mint -> 定义这是什么币
用户 ATA -> 记录该用户持有多少 POINT
mint_to -> 把数量写进去
```

基础部分覆盖了这些操作：

- 创建 Token Mint

- 设置权限

- 关闭 Token Account
- 冻结 / 解冻账户

## 例子：转移代币

假设 Alice 要把 50 POINT 转给 Bob：

1. Alice 需要一个持有 POINT 的 token account
2. Bob 也要有能接收 POINT 的 token account（通常是 ATA）
3. 调用 Token Program 的转账指令

注意：

- 转账的是 token account 之间的余额
- 不是直接在钱包地址之间“转代币”

## 例子：委托交易

Alice 不想直接把私钥交给机器人，但允许机器人最多花 100 POINT。

这时可以：

1. 对 Alice 的 token account 设置 delegate
2. 指定可代理额度

这样机器人在额度内可以操作，但不拥有账户本身。
