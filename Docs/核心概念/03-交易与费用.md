# 交易与费用

来源：

- https://solana.com/zh/docs/core/transactions/index
- https://solana.com/zh/docs/core/fees/index

## 交易是什么

交易是 Solana 的原子执行单元。

一笔交易包含：

- 一条或多条指令
- 签名
- 最近区块哈希

最重要的规则：

- 交易里的任意一条指令失败，整笔交易回滚

## 为什么要把多条指令打包进一笔交易

因为很多业务动作是“要么全成功，要么全失败”。

例如：

1. 创建账户
2. 初始化账户
3. 写入初始数据

如果你把这 3 步拆成 3 笔交易，中间任一步失败都可能留下脏状态。  
打包成一笔交易，就能保持原子性。

## 一个简单例子

```text
交易
  ├─ 指令 1：创建 token account
  ├─ 指令 2：初始化 token account
  └─ 指令 3：向 token account 转入代币
```

如果指令 3 失败，那么前两步也不会最终生效。

## 版本化交易是什么

官方文档提到，Solana 有传统交易和 `V0` 版本化交易。

它的重要意义是：

- 可以结合地址查找表（ALT）
- 减少交易消息里直接放的账户地址数量
- 让复杂交易容纳更多账户

## 费用由什么组成

每笔交易都要付费，费用分两部分：

- `基础费用`
- `优先级费用`

### 基础费用

基础费用主要用于补偿 validator 的签名验证工作。

官方文档里的关键点：

- 按签名数量计费
- 50% 销毁
- 50% 分给 validator

### 优先级费用

优先级费用是你主动加的钱，用于提高交易被当前 leader 优先处理的概率。

文档中的公式可以理解成：

```text
priority_fee = ceil(compute_unit_price * compute_unit_limit / 1_000_000)
```

## 例子：普通转账

一个普通 SOL 转账通常只需要基础费用。  
如果网络不拥堵，你不一定需要加优先费。

## 例子：抢热门 mint / 高频竞争场景

如果你在做：

- 热门 mint
- 抢购
- 高频套利
- MEV 竞争

那你通常会：

- 提高 `compute unit price`
- 适当提高 `compute unit limit`

这样交易更容易被优先打包。

## 开发建议

- 不要盲目把优先费设很高
- 先估算你的交易大概需要多少计算单元
- 复杂交易才考虑额外调高预算

## 常见误区

- 误区 1：交易失败也完全不花钱  
  现实里，失败交易通常仍然会消耗手续费。

- 误区 2：优先费越高越一定成功  
  不是。它只是提高优先级，不保证业务逻辑一定成功。

- 误区 3：多条指令就一定比分开发更便宜  
  不一定，要看总大小、签名数和计算量。
