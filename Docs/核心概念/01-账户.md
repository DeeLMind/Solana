# 账户

来源：

- https://solana.com/zh/docs/core/accounts/index

## 账户是什么

账户是 Solana 存储链上状态的基本单位。你可以把它理解为：

- 地址 = 键
- 账户内容 = 值

每个账户由一个 32 字节地址标识，这个地址可以是：

- 普通公钥
- PDA

## 每个账户的核心字段

官方文档强调，每个账户都共享一套通用结构，理解时重点记住这 5 个字段：

- `lamports`：账户持有的原生余额
- `data`：账户的数据区
- `owner`：谁有权修改这个账户的数据
- `executable`：是否是可执行程序账户
- `rent_epoch`：与租金模型相关的字段

## 账户所有权规则

最重要的一条规则：

- 只有 `owner` 程序能修改账户数据，或者扣减其 lamports

但有个常被忽略的点：

- 任何程序都可以向一个可写账户增加 lamports

## 账户类型怎么理解

可以粗分为几类：

- `程序账户`：保存可执行代码
- `数据账户`：保存程序状态
- `系统账户`：普通钱包/基础账户
- `sysvar`：系统提供的全局只读状态

## 为什么账户要有 rent

Solana 不希望链上被无限堆垃圾数据，所以账户要维持足够余额，才能保持状态存在。

直观理解：

- 账户数据越大
- 需要保持的 lamports 越多

## 例子 1：钱包账户

一个普通钱包地址本质上也是账户。

它最常见的用途是：

- 持有 SOL
- 作为交易签名者
- 支付手续费

## 例子 2：程序状态账户

假设你写一个留言板程序，链上可能有一个 `message_account`：

```text
owner = 你的留言板程序
data = { author, content, updated_at }
```

程序每次执行时：

- 读取这个账户
- 解析 `data`
- 修改 `content`
- 再写回去

## 例子 3：为什么“程序无状态”

很多人第一次学 Solana 会问：

“为什么我不能把变量直接存在程序里？”

因为程序账户主要放的是字节码，不适合保存可变业务状态。  
真正变化的数据，应该放在数据账户里。

## 开发时要记住的坑

- 传给程序的账户不够，程序就没法读写它
- 账户不是你的程序 `owner`，你就不能随便改它的数据
- 账户空间申请太小，后面写入会失败
- 没有保持 rent 免除余额，账户可能无法按预期使用
